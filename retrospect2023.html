Scenario Creator Tool + RiskViz


Building a graphical interface to enhance risk analytics for self driving automobiles







TEAM

TOOLS

DURATION

OVERVIEW

Challenge: Designing an interactive tool that consistently creates editable road scenarios, display play-by-play movement of vehicles, and produce valuable risk analysis reports for self-driving vehicles.

Solution: Calling on UX research methods and Python coding skills to bring interactive Figma prototype to life to ensure successful implementation and reduced user friction.

I worked on both of these projects from start to finish, dedicating over four months to the research, design, and development phases.

BRAINSTORM


Having completed both primary and secondary research, I immediately dove into thinking: what features will most effectively help accomplish the goals I had outlined?


I fixated on choosing the features that aligned with the company's needs and my original goals. The features centralized upon the elements that would allow the interface to have an intuitive experience for the users, promoting constant generation of meaningful risk reports.






INITIAL SKETCHES


I focused on sketching the Risk Visualizer Tool since it was the starting point that would set the precedent for colors, scaling, and overall layout. The first set of sketches centered around incorporating as many components as possible.


However, the second set demonstrates a way to condense information, making elements more realistic and laying out components by how often they are used.




With an idea of how the Risk Visualizer Tool should look, I immediately went to create main screen wireframes while working hand-in-hand with the developers.






WIREFRAMES



I was tasked with both designing and developing the Scenario Creator Tool, using the Python library, Tkinter, to create an intuitive GUI (graphical user interface) from scratch.



Handing the Risk Visualizer Tool to the engineering team, the other UX Design Intern and I worked closely alongside to ensure certain elements were retained and consistent to their original specifications.



I was responsible for translating quantitative data into a visual format, allowing users to understand information efficiently while also preventing it from clouding the screen up with too much information.







USER INTERVIEWS



I recruited the same 3 experts in the auto industry that I surveyed earlier alongside 3 new experts to take part in an interview before developing the final prototype. This would help me verify which components were serving its intended purpose which was to meet the users' needs. 




The users were split into three separate groups - each group having one expert that surveyed and one that was not. They were all given a short demonstration on how the overall flow of the tool worked before being asked to focus on one of the three screens.






How can I incorporate the interview findings into my high fidelity prototype?






ITERATIONS



Although my users were able to understand exactly what was done and could figure out the reasoning behind some designs, a few expressed valuable concerns that I incorporated into my final designs. This was to ensure that my designs were backed with research and user feedback.
